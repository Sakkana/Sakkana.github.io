3AC 三地址码

右侧最多只有一个运算符。

可以充当地址的 Address

  • 变量名 x, y, z
  • 常量 3
  • 编译器生成的临时变量 tmp

常用 3AC 形式

  • x = y bop z
  • x = uop y
  • x = y
  • goto L
  • if x goto L
  • if x rop y goto L

Static Single Assignment (SSA)

给每一个变量一个新的名字。

3AC

$p = a + b\
q = p - c\
p = q * d\
p = d - p\
q = p + q$

SSA

$p_1 = a + b\
q_1 = p - c\
p_2 = q * d\
p_3 = d - p\
q_2 = p + q$

如果是分支结构

  • $if \ e$
    • $x_0$ = 0
    • $x_1 = 1$
  • $y = x + 7$

那么最后的 $x$ 是 $x_0$ 还是 $x_1$ 呢？

这时候会在 merge 的地方引入一个新运算：**$\phi \ 函数$**，如下所示。

  • $if \ e$
    • $x_0 = 0$
    • $x_1 = 1$
  • $x_2 = \phi(x_0, x_1)$
  • $y = x_2 + 7$

SSA 的用处

  • flow-insensitive：不考虑程序执行顺序
    • 精度差，速度快，不考虑执行流
    • 独一无二的变量名可以传递一些 flow 信息，可以获得一定的精度
  • flow-sensitive：考虑程序执行顺序
    • 精度好，速度慢

但是坏处就是会引入太多的变量名和 $\phi \ 函数$。

转换到 SSA 后还是要转换到字节码，会有效率问题。

Basic Block (BB)

BB 是连续的 3AC，并且是最大的序列。

  • 这个 BB 只有一个 entry，并且是第一条指令。不存在第二个 entry。

  • exit 也是唯一的，只能是最后一条指令。

每个 BB 都是 CFG 中的一个结点。

判断 BB 的原则

Control Flow Graphs (CFG)

在 BB 集合划分的基础上添 edge。

在 BB 之间添 edge 的原则

  • There is a conditional or unconditional jump from the end of A to the beginning of B
  • B immediately follows A in the original order of instructions
    • and A does not end in an unconditional jump
