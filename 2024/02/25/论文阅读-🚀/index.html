论文阅读

这个博客记录一下我读过的一些论文，用于准备考研复试 / ra 申请和日常学习。

题目 出处 主题 发表时间 开始阅读 结束阅读
Software-Defined Network Assimilation: Bridging the Last Mile Towards Centralized Network Configuration Management with NAssim SIGCOMM Network manageability; Network management; Computing methodologies 2022 2024.2.25

how to read a paper

分类

  1. 废纸，不值得了解
  2. 值得了解的论文，快速浏览，知道方法和大致思路，不用深究（大部分论文）
  3. 值得精读，有很大启发性和突破性，写作良好，经得住考验
  4. 经典论文，搞清楚方法和细节，自己尝试复现

看结构

八股文结构：

  1. Abstract（开头一小段）
  2. Introduction（1 - 2 页）- 方法、创新点、大致实验效果
  3. Related works（在最后）
  4. Conclusions（结尾）
  5. Approach（具体细节，可能拆分为很多章节）
  6. Experiments（实验）

阅读顺序

原则：不要过于纠结某一句话，侧重理解 author 的想法，切忌从头到尾字斟句酌。

  1. 读 1 - 2 遍 Abstract
    • I. 处理什么问题
    • II. 提出了什么方法和创新点
    • III. 方法取得了什么效果
  2. 跳过中间部分，直接看论文的实验部分
    • I. 实验环境的搭建（数据集、测试方法、评估指标）
    • II. 作者跟什么方法进行比较（本领域景点方法，SOTA (state-of-the-art)）
    • III. 取得了何种定性/定量的效果（消融实验证明有效性）

针对 [分类 2] 中的论文读到这里就可以结束了，对于 [分类 3/4] 可以继续读下去。

  1. 读 Introduction - 更详细版的 Abstract
  2. 读 Approach，细节性的推导和算法实现（对于经典论文慢下来反复看这部分，讲给别人听）
  3. Related Works - 扫一扫，可以跳过
  4. 用审稿人的视角问自己一些问题
    • 作者提出了什么方法和创新点
    • 方法取得了什么效果
    • 方法具有什么局限性和改进之处
