MySql - 8 多表查询

多表查询

1.多表关系

  • 一对多

    案例：部门 - 员工

    关系：一个员工只能归属一个部门，但一个部门可以对应多个员工

    实现：在多的一方建立外键，指向一的一方的主键

  • 多对多

    案例：学生 - 课程

    关系：一个学生可以选修多门课程，一门课程可以供多个学生选择

    实现：建立第三张中间表，中间表至少包含两个外键，分别关联两方的主键

    学生表
    create table student(
    id int auto_increment primary key ,
    name varchar(10),
    no varchar(20)
) comment '学生表';
    课程表
    create table course(
    id int auto_increment primary key,
    name varchar(10)
) comment '课程表';
    中间表
    create table stuent_course(
  id int auto_increment primary key,
  studentid int not null comment '学生id',
  courseid int not null comment '课程id',
  constraint fk_courseid foreign key (courseid) references course (id) ,
  constraint fk_studentid foreign key (studentid) references student (id)
) comment '学生课程中间表';

  • 一对一

    案例：用户 - 用户详情

    关系：一对一关系，多用于单表拆分，将一张表的基础字段放在一张表中，其他详情字段放在另一张表中，以提升操作效率

    实现：在任意一方加入外键，关联另外一方的主键，并且设置外键为唯一的(UNIQUE)

    用户基本信息表

    create table user_basic(
   id int auto_increment primary key comment '主键id',
   name varchar(10),
   age tinyint unsigned,
   gender char(1),
   phone char(11)
) comment '用户基本信息表';

    用户受教育信息

    create table user_edu(
    id int auto_increment primary key  comment '主键id',
    degree varchar(20) comment '学历',
    major varchar(20) comment '专业',
    primary_school varchar(20),
    middle_school varchar(20),
    college varchar(20),
    userid int unique comment '用户id', -- 唯一约束
    constraint fk_userid foreign key (userid) references user_basic(id)
) comment '用户受教育信息';

    2. 多表查询概述

    • 概述：从多张表中查询数据

    • 笛卡尔积：笛卡尔积是指在数学中，两个集合-集合A和集合B所有的组合情况。(在多表查询是，需要消除无效的笛卡尔积)

      假如直接SELECT

      select * from emp, dept;

问题来了，如何消除无效的笛卡尔积？

两个表通过dept_id来建立联系

select * from emp, dept where emp.dept_id = dept.id;

  • 多表查询分类

    连接查询

    ​ 内连接：查询 A ∩ B部分

    ​ 外连接：

    ​ 左外连接：查询 A + (A ∩ B)

    ​ 右外连接：查询 B + (A ∩ B)

    ​ 自连接：当前表与自身的连接连接，自查询必须使用表别名

    子查询

3.连接查询

3.1 内连接

内连接查询两张表的交集部分，即A ∩ B

  • 隐式内连接 (上一节消除无效笛卡尔积的案例就是隐式内连接)
SELECT 字段们 FROM 表1, 表2 WHERE 条件;
  • 显式内连接
SELECT 字段们 FROM 表1 [INNER] JOIN 表2 ON 连接条件;

案例：查询每个员工的姓名与其对应的部门

# 隐式内连接
select e.name, d.id from emp e, dept d where e.dept_id = d.id;

一旦给表起定别名，真实的名字就无法使用了

# 显示内连接
select e.name, d.id from emp e inner join dept d on e.dept_id = d.id;

注意，我么查到了6个人，如下图所示

但实际上数据库中有7个人，还有一个人没有显示的原因是他是实习生，还没有关联部门的外键，因此不属于两张表的交集。

3.2 外连接

  • 左外连接

    查询 A + (A ∩ B)

SELECT 字段们 FROM 表1 LEFT [OUTER] JOIN 表2 ON 条件;

​ 这次就可以将每位员工的数据查询出来了，包括没有与第二行表关联的员工

  • 右外连接

    查询B + (A ∩ B)

SELECT 字段们 FROM 表1 RIGHT [OUTER] JOIN 表2 ON 条件;

通常左外连接用的会多一些，因为右外连接都可以通过颠倒两张表的顺序改成左外连接。

下面两条语句的效果是完全一样的，区别就在于两张表的查询顺序

但实质上都是取部门表的全部数据+部门表与员工表的交集部分

select d.id , e.name from dept d left outer join emp e on e.dept_id = d.id;
select d.id , e.name from emp e left outer join dept d on e.dept_id = d.id;

SELECT 字段们 FROM 表A 别名A JOIN 表A 别名B ON 条件;
  • 自连接 + 内连接

虽然这是在同一张表内查询，但是需要把他们看作两张表进行内连接查询。

但是内连接是取交集，因此如果有员工没有领导就不会被查出来。

  • 自连接 + 外连接

3.5 联合查询

把多次查询的结果合并起来，形成一个新的查询结果集。

查询前提：多条查询返回的字段数量与类型需要相同。

UNION ALL不去重，UNION会去重。

SELECT 字段们 FROM 表A...;
UNION [ALL]
SELECT 字段们 FROM 表B...;

案例：

查询 薪资低于11000 和 年龄大于40 的员工

如果用UNION ALL会出现重复的数据，如果某个人同时满足，此时用UNION就可以去重。
