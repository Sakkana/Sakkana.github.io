子类没有重写时

子类可以调父类的函数，父类不能调子类的函数

重写后，都调子类的函数(father f1 = new son();)

MAIN

import oop.Person;
import oop.Student;

public class MAIN {
    public static void main(String[] args) {

        // Student能调用自己的，或者继承的
        Student s1 = new Student();

        // Person是父类，无法调用子类的
        Person s2 = new Student();  // 父类引用指向子类对象，但不能调用子类独有的函数
        Object s3 = new Student();
        
        Person p = new Person();

        s1.run();
        s2.run();  
        // 在子类Studnet没有写run的时候输出Person run 因为子类Student继承了父类的函数
        // 在子类Student写了run之后s1,s2都输出Son run，因为子类重写了，执行子类
        
        p.run();

        // 能执行哪些函数看左边的类型
        ((Student)s2).eat();   // 这样就可以调用了 高精度转低精度
    }
}

static 变量

在内存中只有一份，被所有的对象共享，可以通过【类名.变量名】的方式调用

package oop;

public class Student extends Person{
    private static int age = 11; // 静态变量，计数
    private double score = 98.9;   // 非静态变量

    public static void main(String[] args) {

        Student s = new Student();

        System.out.println(Student.age);
        System.out.println(s.age);
        System.out.println(s.score);

        // 报错 Non-static field 'score' cannot be referenced from a static context
        // System.out.println(Student.score);
    }
}

抽象类

抽象类不能实例化(new)，被abstract修饰的函数必须被子类实现（子类非abstract）

相当于一个约束

专门用来被继承，相当于种马

没有构造函数

一旦函数里有abstract函数(~virtual Type name() = 0)，这个类就是抽象类，必须用abstract修饰

但是抽象类中可以写普通函数

抽象类

package oop;

public abstract class abs {
    public abstract void func();	// 纯虚函数
    public void func2(){			// 正常函数
        System.out.println("123");
    };
}

子类

package oop;

public class son extends abs{
    @Override
    public void func() {

    }
    public static void main(String[] args) {
        son s = new son();
        s.func2();
    }

接口 interface（变相实现伪多继承）用的不多

不能被实例化 因为接口中没有构造函数

普通类: 只有具体实现函数
抽象类：具体实现函数和规范（纯虚函数）都有
接口：只有规范，自己无法写函数 比抽象类还抽象，专业的约束，只能做约数这件事，约数和实现分离，面向接口编程
package oop;

public interface intf {
    public abstract void f();	// public abstract自动标灰
}
