关于 Higher Order Function

python中函数的泛化(generalization)

C / C++中允许传递指向函数的指针作为参数，赋值给指针变量抑或是作为函数参数。
然而，python中允许将函数名作为变量直接赋值。
这涉及到一种程序设计哲学，将 enviroment 中所有的东西看作一种 value，比如print是预定义的value，它指向了一个在 stdin 中输出东西的函数；def 后的函数名也是一种 value，指向用户自定义的一个函数；用户定义的普通变量则绑定了一个具体的值。
通过将函数名赋值给另一个变量，则该变量也绑定上了该函数，指向该函数。

例如下面这段代码，我们将函数名square和reciprocal作为传入参数传入calculate，被形参term所接收，于是term便先后绑定上了这两个函数，直接调用term()便可以完成这俩函数的功能。

def calcuate(x, term):

print(res)

print(res)

res = calcuate(0, reciprocal)
print(res)

关于柯里化(Currying)

老师的课件上是这么定义的：
The term currying refers to converting a multi-argument function into one
that takes one argument and returns a fcuntion that takes the next argument, and so on.

配套 textbook 是这么写的
We can use higher-order functions to convert a function that takes multiple arguments into a chain of functions that each take a single argument. More specifically, given a function f(x, y), we can define a function g such that g(x)(y) is equivalent to f(x, y). Here, g is a higher-order function that takes in a single argument x and returns another function that takes in a single argument y. This transformation is called currying.

该函数接受一个参数，并且内部又定义了两个匿名函数

def curry2(f):

下面是两个版本的求平方和函数，通过 python Tutor 可以观察执行状态。
这种尾递归的方式层层深入计算，最后一次递归致命一击，计算出一锤定音的结果并一路沿着 frame chain

# 迭代
def sum_squares_iter(N):
    """The sum of k**2"""
    accum = 0
    k = 1
    while k <= N:
        accum += k ** 2
        k += 1
    return accum

# 递归
def sum_squares_recur(N):
    def part_sum(accum, k):
        if k <= N:
            return part_sum(accum + k ** 2, k + 1)
        else:
            return accum

    return part_sum(0, 1)

x = sum_squares_iter(10)
y = sum_squares_recur(10)
print(x, y)

"""

def num_partition(n, k):
    """factors <= k"""
    res = 0
    for i in range(k, 0, -1):
        if n == 0:
            return 1
        elif n < 0:
            return 0
        else:
            res += num_partition(n - i, i) 
    return res

def num_partitions(n, k):
    """Number of distinct ways to express N as a sum of positive
    integers each of which is <= K, where K > 0. (The empty sum is 0.)"""
    if n < 0:
        return 0
    elif k == 1:
        return 1
    else:
        return num_partitions(n - k, k) + num_partitions(n, k - 1)

def count_partitionss(n, m):
    """
    >>> count_partitions(6, 4)
    9
    """
    if n == 0:
        return 1
    elif n < 0:
        return 0
    elif m == 0:
        return 0
    else:
        with_m = count_partitionss(n-m, m)
        without_m = count_partitionss(n, m-1)
        return with_m + without_m

这里对于 start，spare和end的理解至关重要

start:起始的地方
end:目的地
spare:除了start和end之外的那个

经典思路分为四步：
n 为盘子的个数

  1. 若 n == 1,直接 start -> end
  2. 若n > 1，将除了最底下的上面 n-1 盘子挪到空闲地点，即 start -> spare
  3. 将最后一个，即第 n 个盘子挪到终点，即 start -> end
  4. 将空闲地点中的所有盘子移动到目的地

第一步很好懂，第三步可以直接操作，然而 2 和 4 对于程序初心者却听起来匪夷所思。
说得轻巧，直接把上面 n-1 个挪到空闲地点，可是我们一次只能移动一次，该怎么做？
这里就开始递归了。
我们的目标很明确，程序一次只能只能执行一条语句。
此时我们的目标是将 n-1 个盘子挪到空闲盘子，因此递归调用函数，传入 n-1, start_ped, spare_ped。
理解如何传参非常重要。
我们的 start, spare, end 并不是绝对的。
可以将汉诺塔的执行看作一个巨大的状态机，
每一步状态都有自己的 start, spare和end。
对于初始状态来说，即n个盘子都堆叠在原始位置，此时的 start 和 end 就是图中的 1 和 3。
然而将 n-1 个盘子放到空闲地点时，状态转换，进入一层递归，来到一个新的状态机，此时的 start 和 end 分别是 1, 2。
我们需要把每一个状态都是平等的。
每一个状态的个人目的不同(盘子i的st->盘子i的ed)，但最终目的是相同的(n个盘子整体的st->n个盘子整体的ed)。
在状态转换中，盘子可能会从形形色色的 start 被转到形形色色的 end，但是最终目的都是为了达成初始状态的目的： 1 -> 3。
在将 n-1 个盘子挪到空闲地点时，他也是很纯粹的，他不知道自己的命运是什么，他和初始状态一样，只是为了达成 start -> end。
每个人只知道自己的 start 和 end 是什么，不知道别人的是什么。
因此程序就在一次一次的递归中履行自己的使命。
中间的细节一次次地被分解，一次次地递归，直到最后是个原子操作：将一个盘子从属于他的 start 挪到只属于他的 end。
于是乎，故事开始了。
他执行了自己的任务，他的一生结束了。他不知道自己的结束会产生什么影响，因为正如前面所说，每个状态机都是平等的。
然而，程序设计者，以及我们的机器，却有个很大的阴谋。
从每一个原子操作的结束开始，回溯便开始了。
由于只能挪动一个盘子，每一次原子操作的执行都会影响比他多一个盘子的那个状态机的执行。
无数细小的操作只是为了完成一个相对看起来很大的操作，而这个大操作在更大的任务下又显得微不足道。

理解递归程序是个难点，但非常重要。
要充分理解这个程序的整体目标，同时要兼顾子状态的局部目标即兼顾全局性和局部性，也可以看作个人的长远目标和短期目标。而认识到这一点非常重要的是，理解 start 和 end 也具有相对性，对于每一步操作来说他们不一定是相同的标号（即1、2或是3）。但是这个标号对于他们来说便是一生的经历。每个人都一样。

    base-10 representation is the reverse of that of N.
    >>> reverse_digits(0)

总结：其实这个递归层层抽离下去，最后一层叶子节点实际上是最高位，因此返回最高位到倒数第二层后我们要10，层层把位权进上去，因此在递归部分要10，然后考虑当前位是否可以被删掉，如果可以就不用管，如果不可以就要加上最低位

def cycle(f1, f2, f3):
    """Returns a function that is itself a higher-order function.

    >>> def add1(x):
    ...     return x + 1
    >>> def times2(x):
    ...     return x * 2
    >>> def add3(x):
    ...     return x + 3
    >>> my_cycle = cycle(add1, times2, add3)
    >>> identity = my_cycle(0)
    >>> identity(5)
    5
    >>> add_one_then_double = my_cycle(2)
    >>> add_one_then_double(1)
    4
    >>> do_all_functions = my_cycle(3)
    >>> do_all_functions(2)
    9
    >>> do_more_than_a_cycle = my_cycle(4)
    >>> do_more_than_a_cycle(2)
    10
    >>> do_two_cycles = my_cycle(6)
    >>> do_two_cycles(1)
    19
    """
    "*** YOUR CODE HERE ***"
    def set_term(n):
        def get_function(x):
            for i in range(1, n + 1):
                if i % 3 == 1:
                    x = f1(x)
                elif i % 3 == 2:
                    x = f2(x)
                else:
                    x = f3(x)
            return x
        return get_function
    return set_term

def matches(a, b):
    """Return the number of values k such that A[k] == B[k].
    >>> matches([1, 2, 3, 4, 5], [3, 2, 3, 0, 5])
    3
    >>> matches("abdomens", "indolence")
    4
    >>> matches("abcd", "dcba")
    0
    >>> matches("abcde", "edcba")
    1
    >>> matches("abcdefg", "edcba")
    1
    """
    # sum(1 for _ in range(min(len(a), len(b))) if a[_] == b[_])
    # return sum(a[_] == b[_] for _ in range(min(len(a), len(b))))
    return sum([1 for x, y in zip(a, b) if x == y])

关于可变对象和不可变对象

可变对象：list, dictionary, set
不可变对象：tuple, string, int, float, bool

所谓可不可变，指的是这个数据类型本身能否原地改变
如果可以原地改变（id不变），说明是可变的
如果不可以原地改变，而是需要另辟蹊径，在新的地址上重建，则是不可变的

class Product:

    _sales_tax = 0.07   # 成员变量

    def __init__(self, name, price, nutrition_info):
        self._name = name
        self._price = price
        self._nutrition_info = nutrition_info
        self._inventory = 0

    def increase_inventory(self, amount): 
        self._inventory += amount

    def decrease_inventory(self, amount):
        self._inventory -= amount
        if self._inventory < 0:
            print("Oh No!")
    
    def get_nutrition_label(self):
        return "|".join(self._nutrition_info)

    def get_inventory_report(self):
        return f"We have {self._inventory} bars"
    
pizza = Product("fruit pizza", 7.00, ["24 g sugar"])

成员函数只是个普通的函数
