C++11标准 与 陌生用法

最近开始读《C++ Primer》了，记录一些以前C++98中没有遇到过的但是比较优秀的用法以及陌生的知识点。

由于这本书真的太厚了，里面些许知识点是自己已经掌握的，有些新特性与陌生用法是值得自己学习的。然而也不可能每次遇到问题书都在身边，于是将自己觉得有价值的部分记录下来。

因此本文并不是全篇关注C++11，而是新标准与陌生知识相融合的一篇学习记录。

顶层const & 底层const

在这本书里看到了一个新概念：底层const与底层const

顶层const ： 任意对象是常量，指针本身是个常量。

int *const p1 = &a; // 指针本身是常量，不可以再指向别人，长相厮守
const int N = 1e5;	 // 变量N是个const，只读

底层const：指针所指的对象是个const

const int *p2 = &a;	// 不能通过指针修改其指向的对象
const int &ref = &a;	// 声明引用的const的都是底层const

我们还可以见到这种杂合子

const int *const p3 = &a;
// 第一个是底层const，我们无法通过指针修改这个对象
// 第二个是顶层const，长相厮守

执行拷贝操作的时候，底层const会做一些坏事。

拷入和拷出的时候两个数据对象必须具有相同的底层const资格。

  • case 1

这是错误的。

因为p1拥有底层const，我们无法通过指针修改a的值。

而p2却没有底层const，意味着我们可以通过p2修改p2指向的值，即p1指向的值，即a的值。

这与我们的初衷相违背（p1不被允许修改a的值）。

const int *const p1 = &a;
int *p2 = p1;

同理，这样也是不允许的。因为p有可能改变a的值。

const int a = 9;
int *const p = &a;

case 2

这样子是合法的，因为p1和p2都拥有相同的底层const，即p1和p2指向的对象都不能被修改。

const int *const p1 = &a;
const int *p2 = p1;

case 3

这样也是合法的，编译器可以将 int* 转成 const int*

int i = 1;
const int *p = &i;

case 4

这样子是非法的，因为ref有修改i的可能性。

const int i = 99;
int &ref = i;

case 5

这样自然就合法了，因为ref也是只读的。

const int i = 99;
const int &ref = i;

上面这等价于：

const int i = 99, &ref = i;

总而言之，在拷贝的时候，只要关注新定义的变量有没有可能修改拥有底层const的变量就可以了。

constexpr

这是个c++11新增的类型，中文名叫 常量表达式。

constexpr 变量的特点：

  • 值不会改变（这点和const一样）
  • 编译过程就能得到计算结果

using（取别名）

这里介绍的using是普通场合的using，起作用等价于typedef。

auto

让编译器自己推断变量的类型。

case 1

碰到引用，会顺藤摸瓜推断出最原始的值的类型。

相当于 int b = a。

case 2

忽略顶层const（任意对象是常量，指针本身是常量）

如这里的变量a是个常量，但是会被auto忽略掉。

auto出来的b是个普通变量，可以任意读写。

相当于 int b = 1。

case 3

同时碰到引用和顶层const，依然遵循上面的规则。

ri是个引用，引用了int类型的i，然而i是个const，但是顶层const会被忽略，因此b是个可读可写的int。

相当于 int b = i。

case 4

保留底层const （指针指向的内容是const）

变量a是个常量， &a取地址，因此ptr是个指针类型的变量。

由于auto保留底层const，因此ptr无法修改其指向的值。

相当于 const int *ptr = &a。

case 5

如果想要auto推断出来的值是个const，可以显式地告诉编译器。

case 6

声明一个auto的引用时，顶层const会被保留。是否为引用对于auto有重要影响。

由于我们想声明一个变量d作为c的引用，意味着我们可能使用d来修改c的值，然而c是个常量，具有顶层const属性，因此d也必须是一个常量。

因此，我们无法修改d的值，会报错。

case 7

一句auto中的变量需要是同一类型。

总结

  • auto 最关注的是变量的类型，我们只希望获得类型，因此会忽略顶层const， 引用等等。他不希望我们沾染const，引用这种外壳。
  • 但是有些原则是不能触碰的，如果我们auto出来的变量可以修改只读的变量，这是大忌。因此auto不会忽略底层const。
  • 如果我们真的希望auto出来的变量确实是个const或者引用，那么我们就明确地告诉编译器。和普通变量命名规则一样，大家要么都是const要么都可读可写，一根绳上的蚂蚱。

上面三条记牢，在用auto的时候就会少犯一些错你。

decltype

这个也是c++11除了auto之外引入的第二个新的类型说明符。

他会根据括号内的操作数推断类型并返回。

在此过程中，编译器推断表达式的类型但是不会计算它的值。

然,decltype处理顶层const和引用和auto有些许不同。

case 1

如果要推断一个变量，这个变量是啥就是啥，因为我们希望得到的结果和里面的变量一模一样。

顶层const和引用它都会考虑进去。看起来比auto更加聪明也更危险。

这个例子中，i拥有顶层const属性，因此decltype返回的也是个 const int，因而 a 不可以被修改。

这里的 j 是个引用类型，与 i 绑定，因此修改 j 就是在修改 i。

我们使用 decltype(j)，它会返回和 j 一样的数据类型，即int &。

因此这条语句相当于 int &rj = j。

从而我们又将 rj 与 j 绑定。

从这里开始， i, j, rj 都是同一根绳子上的蚂蚱，牵一发而动全身。

编译器怕我们太笨了也会告诉我们，这个 rj 是个引用变量，因此需要马上初始化。

如果不绑定一个变量他就会报错。

case 2

如果decltypr使用的表达式不是一个变量，那么它会返回表达式结果相对应的类型。

然而引用的代价太大了，有时候我们单纯只是像获得类型，就行auto那样。

下面这个表达式 j + 0 会返回一个 int 类型的值，因此decltyp返回的就是一个普通的 int。

case 3

奇怪的是，对地址的解引用操作也会返回引用类型。

如下面的 decltype(*ptr) 就会返回 int &，因此我们修改 ref 就是在修改 a。

case 3

decltype(a) 和 decltype((a)) 是不一样的

不加括号就和上面的规则一样。

相当于 int b = a;。

如果加上了一层括号，编译器会返回引用类型。

相当于 int &b = a;

内置数组初始化vector

曾经的C标准和C++标准不允许使用内置数组的元素直接初始化别的数组，也不允许使用vector初始化内置数组。

新标准允许使用内置数组初始化vector了。

内置数组使用迭代器

后面会细说。

这里要介绍的是新标准中，允许内置数组使用迭代器了。

其中,begin 返回第一个元素的指针，end返回最后一个尾后指针。

更一般地，使用vector<T> v(&arr[i], &arr[j])便可以用下标区间在[i, j)之间的元素初始化vector了。

多维数组遍历

#include <iostream>
#include <vector>
#include <iterator>
using namespace std;

int main()
{
    int cnt = 0;

    int ia[3][7];
    for(int (&i)[7] : ia)
        for(int &j : i)
        {
            j = ++ cnt;
            cout << j << endl;
        }

    for(size_t i = 0; i < sizeof(ia) / sizeof(ia[0]); ++ i)
        for(size_t j = 0; j < sizeof(ia[0]) / sizeof(ia[0][0]); ++ j)
            cout << ia[i][j] << endl;

    for(int (*i)[7] = ia; i != ia + 3; ++ i)
        for(int *j = *i; j != *i + 7; ++ j)
            cout << *j << endl;

    return 0;
}

pass by reference 还是 by value？

对于占用内存非常大的对象我们常常传它的引用作为函数参数。

函数只能 return 一个参数，我们可以通过传引用把其他的返回值通过引用的作为函数参数传入。其实这个在C中也经常用到，比如 pthread_create中传入的第一个参数就是个传出参数，像个钩子可以带出来子线程的tid。

侯捷老师：传引用很快，形式也很漂亮。C语言中擅长传递指针，也很轻量级。所有传参都传引用。如果不想要改 value ，就 pass by reference to const。

return by reference 还是 by value？

返回尽量by reference。有些情况下不可以。

千万不要返回局部变量的reference。

this指针

每个对象内部都有一个隐式参数—this指针。

这个指针的地址指向自己，也就是说，他保存的是自己的首地址。

#include <iostream>
using namespace std;

class Elephant{
public:
    int longNose = 1, fatLeg = 2, greatTail = 3;
    double twoEyes = 4.4;
    void *get_address()
    {
        return this;
    }
};

int main()
{
    Elephant a;

    cout << "&a: " << &a << endl;
    cout << "this: " << a.get_address() << endl;

    cout << &a.longNose << endl;
    cout << &a.fatLeg << endl;
    cout << &a.greatTail << endl;
    cout << &a.twoEyes << endl;

    return 0;
}

可以看到 this的值 = 对象a的首地址。

成员函数形参参数列表后面的const

    return 0;

如果你尝试这么做，编译器会报错。

因为你在参数列表后指定了const，表面不可以该改变this指针指向的对象。

this指向一个常量。

如果我定义了一个 const 的对象，并且调用某个成员函数，该成员函数内部修改了成员变量的值，就会报错。

同一个 class 实例化的 objects 互为 friends

#include <iostream>

class complex
{
public:
    complex(double _r, double _i)
        : re(_r), im(_i)
    { }

    double func(const complex& param)	// 这是完全合法的
    { return param.re + param.im; }
 
private:
    double re, im;
};

int main()
{
    complex c1(19.2, 10.1), c2(90.1, 0.2);

    std::cout << c1.func(c2);

    return 0;
}
