Linux 多进程

1. 什么是进程

1.1 程序 & 进程

元信息：文件大小，创建、修改信息等

程序只占用磁盘资源，是死的；进程占用内存资源、CPU资源，是活的。

1.2 单道 & 多道程序设计

早期计算机，一个计算机只能运行一个程序，如果要运行别的程序，当前程序必须停止，效率低下。

在单任务CPU中， 多道程序就是在管理程序的控制下穿插执行，而不是同时执行。目的：提高CPU利用率。

1.3 时间片

把时间切成一片一片。

进程调度的程序来切换进程，保存（放到PCB中，见1.5）原先进程的状态，再加载另一个进程的状态。

时间片太短，会导致切换在整个程序运行中占了很大的百分比；如果时间片太长，会导致人能感觉出来，宏观上无法同时执行。

1.4 并行 & 并发

并发问题：双11时，许多人会同时访问服务器，如何保证服务器在同一时刻大量的同时访问不崩溃掉？

1.5 进程控制块 (PCB)

（当前工作目录）虚拟地址空间中有个环境变量

利用 ulimit -a 可以查看使用资源上线(可修改，但一般不这么做)

2. 进程状态转换

2.1 进程的状态

就绪态时，OS已经为进程分配了内存资源，但是没有CPU资源。

简而言之，万事俱备，等待调度，只剩被CPU执行。

阻塞态无法直接变成运行态，而是需要先进入就绪态。

查看进程快照snapshot（静态数据）

3. 进程创建

NAME
       fork - create a child process

SYNOPSIS
       #include <sys/types.h>
       #include <unistd.h>

       pid_t fork(void);	// pid_t 是个 int

DESCRIPTION
       fork()  【creates  a  new  process】 by 【duplicating the calling process】.  The new process is referred to as the child process.
       The calling process is referred to as the parent process.
测试代码
#include <unistd.h>
#include <sys/types.h>
#include <stdio.h>

int main()
{
    pid_t pid = fork();

    if(pid == 0)
    {
        printf("child process, return value=%d, pid=%d, ppid=%d\n",\
                pid, getpid(), getppid());
    }
    else if(pid > 0)
    {
        printf("parent process, return value=%d, pid=%d, ppid=%d\n",\
                pid, getpid(), getppid());
    }
    else
    {
        perror("fork");
        return -1;
    }

    for(int i = 1; i <= 1000; ++ i)
    {
        printf("%d, pid = %d\n", i, getpid());
        //sleep(1);
    }

    return 0;
}
我们无法揣测子进程和父进程谁先执行

所有的代码在父子进程的地址空间中都有，只不过在子进程中，

调用fork()之前的代码不会被执行。

这一段非常明显，因为我们发现 父进程pid=153207，子进程pid=153208

父进程首先运行，打印到684之后中断发生，

子进程开始运行，打印一行消息，

之后再次切换到父进程打印685，

之后又切换到子进程，打印1，2

如此循环往复，不断切换…

具体谁执行却决于操作系统调度哪个进程

3. 父子进程

3.1 父子进程の虚拟地址空间情况

The child process and the parent process run in 【separate memory spaces】.  At the time of fork() both memory spaces have the 【same content】.  Memory writes, file mappings (mmap(2)), and unmappings (munmap(2)) performed by one of the processes do not affect the other.

       The child process is an exact duplicate of the parent process except for the following points:

		*  The child has its 【own unique process ID】, and this PID does not match the ID of any existing  process  group (setpgid(2)) or session.

       *  The child's parent process ID is the same as the parent's process ID.

       *  The child does 【not inherit】 its parent's 【memory locks】 (mlock(2), mlockall(2)).

       *  Process  resource  utilizations  (getrusage(2))  and  CPU time counters (times(2)) are reset to zero in the child.

       *  The child's set of pending signals is initially empty (sigpending(2)).

       *  The child does not inherit semaphore adjustments from its parent (semop(2)).

       *  The child does not inherit process-associated record locks from its parent (fcntl(2)).  (On the other hand, it does inherit fcntl(2) open file description locks and flock(2) locks from its parent.)

       *  The child does not inherit timers from its parent (setitimer(2), alarm(2), timer_create(2)).

       *  The  child  does  not  inherit  outstanding  asynchronous  I/O  operations  from  its  parent (aio_read(3), aio_write(3)), nor does it inherit any asynchronous I/O contexts from its parent (see io_setup(2)).

父进程和子进程的虚拟地址空间完全一样，包括用户区和内核区。（除了两者的pid不同，以及fork的返回值不同（父进程中返回值=子进程pid，子进程中返回值=0））。

因此修改同名变量并不会影响对方，因为两者分明就在不同的内存中，只不过数据都是赋值过来的罢了。

测试代码
#include <unistd.h>
#include <sys/types.h>
#include <stdio.h>
#include <stdlib.h>

int global_a = 0;	// 测试全局变量（子进程复制）

int main()
{
    int local_a = 99;	// 测试局部变量（子进程复制）

    pid_t pid = fork();

    int after_fork_a = 999;	// 测试fork之后各自产生的局部变量

    printf("Before: global_a=%d, local_a=%d, after_fork_a=%d\n",\
            global_a, local_a, after_fork_a);

    if(pid == 0)
    {
        printf("child process, return value=%d, pid=%d, ppid=%d\n",\
                pid, getpid(), getppid());
        
        ++ global_a, ++ local_a, ++ after_fork_a;
        
        printf("global_a=%d, local_a=%d, after_fork_a=%d\n",
                global_a, local_a, after_fork_a);
    }
    else if(pid > 0)
         else if(pid > 0)
    {
        printf("parent process, return value=%d, pid=%d, ppid=%d\n",\
                pid, getpid(), getppid());
             
        -- global_a, -- local_a, -- after_fork_a;
             
        printf("global_a=%d, local_a=%d, after_fork_a=%d\n",
                global_a, local_a, after_fork_a);
    }
    else
    {
        perror("fork");
        return -1;
    }

    return 0;
}

写时复制

实际上，Linux的fork()使用的是写时拷贝（copy-on-write）技术。

写时拷贝是一种可以推迟甚至避免拷贝数据的技术。

内核此时并不复制整个进程的地址空间，而是让父子进程共享同一个地址空间。

只有在需要写如的时候才会复制地址空间，从而使各个进程拥有各自的不同的地址空间。

换而言之，资源的复制是在需要写入的时候才会进行，

在此之前通过只读（read-only）方式共享数据。

因为如果你没有写操作，拷贝会拉跨性能，

更别说你fork完了就直接exec。

fork()之后父子进程共享文件，

fork()产生的子进程与父进程有相同的文件描述符，指向相同的文件，

引用计数增加，共享文件偏移指针。

读的时候共享，一旦有写操作就马上复制，互不干扰

3.2 父子进程の关系 と GDB多进程调试

父子进程关系

区别：1.fork()函数的返回值不同

​ 父进程：> 0 返回子进程的PID

​ 子进程：= 0

​ 2.PCB（Process Control Block）中的一些数据

​ 当前进程的ID -> PID

​ 当前进程的父进程ID -> PPID

​ 信号集

共同点：

  1. 某些状态下，子进程刚被创建出来，还没有执行任何写数据的操作时，

    ​ 用户区数据相同

    ​ 文件描述符表相同

      2. 父子进程`读时共享`变量，写时候`拷贝`变量（因此不能用变量进行进程间通信）

4. exec 函数族

函数族：一系列功能相近的函数，类似C++中的函数重载

由于直接调用exec会覆盖当前进程的数据，而主进程往往也要做一些事情，因此我们先fork出来一个子进程，在子进程中执行exec。

exec执行成功后不返回，因为本身进程中的内容被替换了（除了PID等表面信息），执行exec的进程中的代码全都变了，变成了新程序的代码，因此返回没有任何意义。所以只有调用失败时才会返回，告知调用失败，原来的代码不会被替换，从exec的调用点接着往下执行。

只有 execve 才是UINX/LINUX的系统函数，其余6个都是在这个基础上的封装，其中 execl和execlp比较常用。

execl
int execl(const char *pathname, const char *arg, ... //(可变参数，一个或多个)
                      /* (char  *) NULL */);

参数：

  • 文件路径（文件路径及建议使用绝对路径）
  • 传入参数列表
    • argv[0]时程序的名称，后面都是参数列表，最后以NULL结束（哨兵，是程序知道解析完成）

返回值：

  • 只有在调用失败时才会返回 -1，并设置 errorno
  • 调用成功没有返回值
测试代码
#include <unistd.h>
#include <stdio.h>

int main()
{
    pid_t pid = fork();

    if(pid > 0)
    {
        printf("Parent process, pid = %d\n", getpid());
    }else if(pid == 0)
    {
        execl("hello", "hello", NULL);
        
        printf("This might not be printed.\n");	// 这句话是不应该被输出的
    }

    for(int i = 0; i < 3; ++ i)
    {
        printf("i = %d, pid = %d\n", i, getpid());
    }

    return 0;
}

其中，可执行程序hello的源代码为

1 #include <stdio.h>                                                                                                                 2
3 int main()
4 {
5     printf("Hello");
6     return 0;
7 }

运行成功，可以看到execl后面的那句printf("This might not be printed.\n");和后面的循环并没有被执行，可见新的可执行程序将原来的子进程代码替换掉了。

然而，这里的Hello似乎没有和其他的输出在同一个块中，这是因为孤儿进程的缘故。

只要在原来的代码父进程中加一句sleep，就可以解决这个问题。

此外，我们也可以执行系统的可执行程序

测试代码（执行 ps aux）
#include <unistd.h>
#include <stdio.h>

int main()
{
    pid_t pid = fork();

    if(pid > 0)
    {
        printf("Parent process, pid = %d\n", getpid());
        sleep(1);
    }else if(pid == 0)
    {
        int ret = execl("/bin/ps", "ps", "aux", NULL);	// 第一个参数写./ps也可以
        
        if(ret == -1)
            perror("execl");
        
        printf("This might not be printed.\n");
    }

    for(int i = 0; i < 3; ++ i)
    {
        printf("i = %d, pid = %d\n", i, getpid());
    }


    return 0;
}

#include <unistd.h>
#include <stdio.h>

int main()
{

}

5. 进程退出 と 孤儿进程 と 僵尸进程

子进程退出后，父进程可以获得子进程的退出状态，因为什么原因退出。

子进程的用户区数据会自动释放，父进程有义务回收内核区的数据（PCB）。

exit (标准C库函数)

标准C库的exit()会做更多的事情（相比_exit()）,包括刷新缓冲区（将缓冲区中的数据写入相应的文件）等，最后调用底层的_exit()。

测试代码
#include <unistd.h>
#include <stdlib.h>
#include <stdio.h>

int main()
{
    printf("Hello\n");
    printf("world");

    exit(0);
}

\n会刷新缓冲区，因此在打印了Hello之后缓冲区就空了，之后再打印world，这个实际上也先被写入了缓冲区，调用exit()退出时，系统也先刷新缓冲区，world被写在了屏幕上。

_exit (Linux系统函数)
测试代码
#include <unistd.h>
#include <stdlib.h>
#include <stdio.h>

int main()
{
    printf("Hello\n");
    printf("world");

    _exit(0);
}

神奇的是，world并没有被输出，这是因为前面\n刷新了缓冲区，这句printf执行完之后缓冲区被刷新了，之后的world被写入了缓冲区，但是_exit()系统函数并不会刷新缓冲区，因此只输出了Hello。

孤儿进程

测试代码
#include <unistd.h>
#include <sys/types.h>
#include <stdio.h>
#include <stdlib.h>


int main()
{
    pid_t pid = fork();


    if(pid == 0)
    {
        sleep(1);	// 子进程睡1s，父进程早就运行完死了

        printf("child process, pid=%d, ppid=%d\n",\
                getpid(), getppid());

        for(int i = 0; i < 3; ++ i) printf("%d\n", i);
    }
    else if(pid > 0)
    {

        printf("parent process, pid=%d, ppid=%d\n",\
                getpid(), getppid());

        for(int j = 0; j < 3; ++ j) printf("%d\n", j);
    }
    else
    {
        perror("fork");
        return -1;
    }

    return 0;
}

可见，父进程运行结束后，shell马上给出了提示符，但是子进程运行完没有给。

这是因为，当我们./app开始运行，shell会自动切换到后台运行，有输出了再切换到前台输出。我们的shell是这个pid=157394的父进程，他知道这个进程结束了，因此shell切换回了前台，显示提示符。但是他不知道我们运行的这个进程还有个子进程。

由于子进程和父进程共享文件描述符，因此子进程也会输出到终端。

最终子进程被1号进程init领养，回收。

孤儿进程没有任何危害。

僵尸进程

孤儿进程是父亲先死亡，孩子变成了孤儿，被init释放。

僵尸进程是孩子死亡，但是父亲没有去回收子进程的资源。为什么会这样呢？

示例代码
#include <unistd.h>
#include <sys/types.h>
#include <stdio.h>
#include <stdlib.h>

int main()
{
    pid_t pid = fork();


    if(pid == 0)
    {

        printf("child process, pid=%d, ppid=%d\n",\
                getpid(), getppid());

    }
    else if(pid > 0)
    {
        while(1)
        {
            printf("parent process, pid=%d, ppid=%d\n",\
                getpid(), getppid());
            sleep(1);
        }

    }
    else
    {
        perror("fork");
        return -1;
    }

    for(int i = 0; i < 3; ++ i) printf("%d\n", i);

    return 0;
}

子进程结束后，父进程仍被困在死循环里没有去回收子进程，因此子进程变成了一个僵尸进程。

且无法用 kill -9 杀死。

如果产生了大量的僵尸进程，会导致大量的进程号被占用。

6. wait 函数 と waitpid 函数

进程回收

父进程通过调用wait 或 waitpid 得到子进程的退出状态，并彻底清除这个进程。

wait是阻塞地等待，如果子进程一直没结束，父进程就一直阻塞，直到子进程结束。

子进程死亡后父进程中的wait不阻塞，往下执行。

waitpid 可以设置为不阻塞，还可以指定等待哪个子进程结束。

应该在循环中多次调用 wait，清理多个子进程。

wait

执行成功返回被回收的子进程PID，没有子进程了或失败返回 -1。

示例代码（僵尸进程的产生）
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>
#include <stdlib.h>

int main()
{
    pid_t pid;

    for(int i = 0; i < 5; ++ i)
    {
        pid = fork();
        if(pid == 0)
            break;
    }

    
    if(pid == 0)
    {
        printf("I'm son, pid=%d\n", getpid());
        sleep(1);

        printf("son pid=%s died\n", getpid());
        exit(0); 
    }else if(pid > 0)
    {
        while (1)
        {
            printf("I'm parent, pid=%d\n", getpid());
            sleep(1);
        }
    }else
    {
        perror("fork");
        return -1;
    }


    return 0;
}

可以看到，执行程序之后，，5个子进程打印了各自的进程号之后直接死亡，父进程继续执行。

由于父进程一直在死循环中执行，没有去回收子进程，导致已经死亡却没有被回收的子进程成为了5个僵尸进程，使用 ps aux 可以看到他们的状态是Z。

当我们用 kill -9 去杀死僵尸进程，惊奇地发现他们无法被杀死。

kill 之后这个进程仍然存在。我们只能通过 ctrl + c 来中止当前的父进程。

示例代码（父进程通过wait释放子进程资源）
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>
#include <stdlib.h>

int main()
{
    pid_t pid;

    for(int i = 0; i < 5; ++ i)
    {
        pid = fork();
        if(pid == 0)
            break;
    }

    
    if(pid == 0)
    {
        while(1)
        {
            printf("I'm son, pid=%d\n", getpid());
            sleep(1);
        }

        exit(0); 
    }else if(pid > 0)
    {
        while (1)
        {
            printf("I'm parent, pid=%d\n", getpid());

            int ret = wait(NULL);	// 父进程会在这里阻塞
            printf("child pid=%d died!\n", ret);

            sleep(1);
        }
    }else
    {
        perror("fork");
        return -1;
    }


    return 0;
}

可以看到，父进程就打印了一行就再也没有出现过了，因为wait是阻塞等待，父进程阻塞在wait那里等待子进程的死亡。

此时父进程和他的5个子进程都是正常运行的进程。

接下来，我们将5个子进程挨个杀掉。

可以看到输出变化了，这是因为父进程通过wait阻塞等待，终于等到一个子进程死亡，他马上为子进程办理后事，轻装上阵，因此打印出了 I’m parent…

然而这里好像有点bug，wait若执行成功会返回子进程的PID给ret并打印，然而这里打印的都是父进程的PID，啥时候我再研究一下。

查看子进程退出状态
示例代码（子进程自然死亡）
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>
#include <stdlib.h>

int main()
{
    pid_t pid;

    for(int i = 0; i < 5; ++ i)
    {
        pid = fork();
        if(pid == 0)
            break;
    }

    if(pid == 0)
    {
        printf("I'm son, pid=%d\n", getpid());
        sleep(1);
        
        printf("child %d died\n", getpid());
        exit(0); 
    }else if(pid > 0)
    {
        while (1)
        {
            printf("I'm parent, pid=%d\n", getpid());

            int st;
            int ret = wait(&st);
            if(ret == -1) break;

            if(WIFEXITED(st)) printf("child died peacefully, status:%d\n", WEXITSTATUS(st));        // 正常中止
            if(WIFSIGNALED(st)) printf("child was killed by signal %d\n", WTERMSIG(st));      // 被信号中止

            sleep(1);
        }
    }else
    {
        perror("fork");
        return -1;
    }


    return 0;
}

父进程执行了一句话之后就阻塞，等待子进程的退出。

5个子进程都正常退出，父进程接收到的退出状态也因此是0.

关于wait的函数参数 int *wstatus

这是个传出参数，因此要传入指针，获取子进程的退出状态。

waitpid

pid_t waitpid(pid_t pid, int *wstatus, int options);

参数：

  • 指定的PID
    • PID > 0：指定谁就回收谁
    • PID = 0：回收在该进程组中的所有子进程
    • PID = -1：回收所以子进程（包括其他组），相当于wait（最常用）
    • PID < -1：回收某个进程组的子进程，GID = ABS(PID)
  • 子进程退出状态
  • 设置阻塞/非阻塞
    • 0：阻塞等待
    • WNOHANG：非阻塞，没有子进程退出就立马返回，不等待

返回值：

  • > 0：返回子进程PID
  • = 0：当options = WNOHANG，表示还有子进程活着
  • = -1：没有子进程了，或执行错误
示例代码（阻塞状态）
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>
#include <stdlib.h>

int main()
{
    pid_t pid;

    for(int i = 0; i < 5; ++ i)
    {
        pid = fork();
        if(pid == 0)
            break;
    }

    if(pid == 0)
    {
        while(1)
        {
            printf("I'm son, pid=%d\n", getpid());
            sleep(1);
        }
        printf("child %d died\n", getpid());
        exit(0); 
    }else if(pid > 0)
    {
        while (1)
        {
            sleep(1);
            printf("I'm parent, pid=%d\n", getpid());

            int st;
            int ret = waitpid(-1, &st, 0); 
            if(ret == -1) break;
            //if(ret == 0) continue;

            if(WIFEXITED(st)) printf("child died peacefully, status:%d\n", WEXITSTATUS(st));        // 正常中止
            if(WIFSIGNALED(st)) printf("child was killed by signal %d\n", WTERMSIG(st));      // 被信号中止

            
        }
    }else
    {
        perror("fork");
        return -1;
    }


    return 0;
}

子进程一直在跑，父进程一直在阻塞等待，只输出了一句话。

示例代码（非阻塞）
#include <stdio.h>
#include <unistd.h>
#include <sys/wait.h>
#include <stdlib.h>

int main()
{
    pid_t pid;

    for(int i = 0; i < 5; ++ i)
    {
        pid = fork();
        if(pid == 0)
            break;
    }

    if(pid == 0)
    {
        while(1)
        {
            printf("I'm son, pid=%d\n", getpid());
            sleep(1);
        }
        printf("child %d died\n", getpid());
        exit(0); 
    }else if(pid > 0)
    {
        while (1)
        {
            sleep(1);
            printf("I'm parent, pid=%d\n", getpid());

            int st;
            int ret = waitpid(-1, &st, WNOHANG); 
            if(ret == -1) break;
            if(ret == 0) continue;	// 没有子进程死亡，父进程继续跑

            if(WIFEXITED(st)) printf("child died peacefully, status:%d\n", WEXITSTATUS(st));        // 正常中止
            if(WIFSIGNALED(st)) printf("child was killed by signal %d\n", WTERMSIG(st));      // 被信号中止

            
        }
    }else
    {
        perror("fork");
        return -1;
    }


    return 0;
}

可以看到，在非阻塞状态下，父进程和子进程同时在运行。
